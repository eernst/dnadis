# Output File Formats

This document provides detailed documentation for all output files generated by `final_finalizer`.

For information on log files, see the `--log-file` option in the main [README](../README.md#common-options).

## Overview

### TSV Data Files

| File | Description |
|------|-------------|
| `*.contig_summary.tsv` | Per-contig classification with metrics |
| `*.evidence_summary.tsv` | Synteny evidence per contig √ó reference chromosome |
| `*.segments.tsv` | Individual synteny segments (protein alignments) |
| `*.macro_blocks.tsv` | Aggregated synteny chains/macro-blocks |
| `*.ref_lengths.tsv` | Reference chromosome lengths |
| `*.contaminants.tsv` | Detailed contaminant summary with taxonomic lineage (if contaminants detected) |

### Visualization Files (when --plot enabled)

| File | Description |
|------|-------------|
| `*.chromosome_overview.pdf` | Multi-panel synteny visualization showing contig composition and alignment identity |
| `*.depth_overview.pdf` | Read depth visualization by classification and chromosome (if `--reads` provided) |
| `*.depth_overview.html` | Interactive depth visualization with tooltips (if `--plot-html` and `--reads` provided) |
| `*.contaminant_treemap.pdf` | Phylogenetic breakdown of contaminants as hierarchical treemap (if contaminants detected) |
| `*.contaminant_table.html` | Interactive table of top contaminants ranked by abundance (if contaminants detected) |

## contig_summary.tsv

The main output file containing classification results and quality metrics for each query contig.

### Classification Columns

| Column | Type | Description |
|--------|------|-------------|
| `contig` | string | New contig name (reflects chromosome assignment, e.g., `chr5A` or `contig_1`) |
| `original_name` | string | Original contig name from input FASTA |
| `classification` | string | Classification category (see [Categories](#classification-categories)) |
| `classification_confidence` | string | Confidence level: `high`, `medium`, or `low` (see [Confidence](#classification-confidence)) |
| `reversed` | yes/no | Whether the contig was reverse-complemented to match reference orientation |
| `contaminant_taxid` | integer | NCBI taxonomy ID (only for `contaminant` classification) |
| `contaminant_sci` | string | Scientific name of contaminant organism |

### Evidence Strength Columns

These columns provide the underlying evidence used to determine classification confidence.

| Column | Type | Description |
|--------|------|-------------|
| `gc_content` | float | GC content of contig (0.0-1.0) |
| `gc_deviation` | float | How many standard deviations the contig GC differs from baseline (reference GC for chrom_assigned, assembly chromosome GC for others) |
| `synteny_score` | float | Synteny evidence strength (0.0-1.0); based on gene proportion |
| `contam_score` | float | Contaminant evidence strength (0.0-1.0); normalized centrifuger score |
| `contam_coverage` | float | Fraction of contig covered by contaminant alignments (0.0-1.0) |

### Read Depth Columns

These columns are populated when `--reads` is provided. They contain per-contig read depth statistics computed by mosdepth.

| Column | Type | Description |
|--------|------|-------------|
| `depth_mean` | float | Mean read depth across the contig (computed by mosdepth) |
| `depth_median` | float | Median read depth across the contig |
| `depth_std` | float | Standard deviation of read depth across windows |
| `depth_breadth_1x` | float | Fraction of bases with ‚â•1x coverage (0.0-1.0) |
| `depth_breadth_10x` | float | Fraction of bases with ‚â•10x coverage (0.0-1.0) |

These depth metrics are useful for:
- **Copy number assessment**: Chromosomes should have ~1x relative depth; debris may show elevated depth
- **Misassembly detection**: Abrupt depth changes may indicate chimeric regions
- **Quality control**: Low breadth indicates incomplete coverage

### Assignment Columns

| Column | Type | Description |
|--------|------|-------------|
| `assigned_subgenome` | string | Subgenome identifier (e.g., `A`, `B`, `At`, `Dt`) or `NA` if none |
| `assigned_ref_id` | string | Best-matching reference chromosome ID (e.g., `chr5A`) or `NA` |
| `assigned_chrom_id` | string | Chromosome number without subgenome (e.g., `chr5`) |
| `status` | string | Assignment status: `OK`, `NO_HITS`, `AMBIG_LOW_FRAC`, or `AMBIG_LOW_RATIO` |

### Size and Coverage Columns

| Column | Type | Description |
|--------|------|-------------|
| `length` | integer | Contig length in base pairs |
| `best_ref_union_bp` | integer | Total aligned base pairs to best reference (union of all alignments) |
| `best_ref_union_frac` | float | Fraction of contig covered by alignments to best reference (`best_ref_union_bp / length`) |
| `total_aligned_bp` | integer | Total aligned base pairs across all reference chromosomes |
| `n_ref_hits` | integer | Number of distinct reference chromosomes with alignments to this contig |

### Scoring Columns

| Column | Type | Description |
|--------|------|-------------|
| `best_score` | float | Synteny score for best reference assignment |
| `second_score` | float | Synteny score for second-best reference |
| `score_ratio` | float | Ratio of best to second score (`best_score / second_score`); `Inf` if no second hit |

### Gene Statistics Columns

| Column | Type | Description |
|--------|------|-------------|
| `ref_gene_proportion` | float | Proportion of reference chromosome genes aligned to this contig (0.0-1.0). Higher values indicate stronger synteny support. |
| `genes_per_Mbp` | float | Gene density: aligned reference genes per megabase of query sequence. Computed as `gene_count / (contig_length_bp / 1,000,000)`. |

### Quality Flag Columns

| Column | Type | Description |
|--------|------|-------------|
| `chimeric` | yes/no | Whether the contig appears to be chimeric (significant alignments to multiple chromosomes) |
| `chimera_reason` | string | Details of chimera detection (e.g., `primary_frac=0.650,secondary_frac=0.250`) |
| `low_coverage` | yes/no | Whether total aligned bp is below `--chr-like-minlen` threshold |

### Best Reference Alignment Metrics

| Column | Type | Description |
|--------|------|-------------|
| `best_matches` | integer | Total matching bases in alignments to best reference |
| `best_aln_len` | integer | Total alignment length to best reference |
| `best_identity` | float | Alignment identity to best reference (`best_matches / best_aln_len`) |
| `best_distance` | float | Alignment distance to best reference (`1 - best_identity`) |

### Second-Best Reference Alignment Metrics

| Column | Type | Description |
|--------|------|-------------|
| `second_ref_id` | string | Second-best reference chromosome ID or `NA` |
| `second_ref_union_bp` | integer | Total aligned base pairs to second-best reference |
| `second_matches` | integer | Total matching bases in alignments to second-best reference |
| `second_aln_len` | integer | Total alignment length to second-best reference |
| `second_identity` | float | Alignment identity to second-best reference |
| `second_distance` | float | Alignment distance to second-best reference |

### Classification Categories

| Category | Description |
|----------|-------------|
| `chrom_assigned` | Chromosome-length contig assigned to a reference chromosome |
| `chrom_unassigned` | Chromosome-length contig without reference assignment (novel or failed synteny gates) |
| `organelle_complete` | Complete organelle genome (chrC or chrM) |
| `organelle_debris` | Partial organelle sequence |
| `rDNA` | Ribosomal DNA repeat unit |
| `contaminant` | Sequence from contaminating organism |
| `chrom_debris` | High-coverage (‚â•80%), high-identity (‚â•90%) duplicate of an assembled chromosome |
| `debris` | Assembly debris with reference homology (‚â•50% coverage) or protein hits (‚â•2) |
| `unclassified` | Could not be classified (below chromosome-length threshold) |

### Classification Confidence

The `classification_confidence` column indicates how certain the classification is, based on multiple lines of evidence.

| Level | Criteria |
|-------|----------|
| `high` | Strong, consistent evidence supporting the classification |
| `medium` | Moderate evidence, or some conflicting signals |
| `low` | Weak evidence, conflicting signals, or unusual characteristics |

**Evidence factors considered:**

- **Gene proportion**: Fraction of reference chromosome genes aligned to the query contig
- **GC deviation**: How many standard deviations (œÉ) the contig's GC content differs from a GC baseline:
  - For `chrom_assigned`: compared to reference nuclear genome GC
  - For all other categories: compared to assembly chromosome GC (more appropriate for divergent genomes)
- **Coverage**: Fraction of contig covered by alignments (0.0-1.0)
- **Identity**: Alignment identity (0.0-1.0)
- **Protein hits**: Number of unique reference protein-coding genes with miniprot alignments

**Per-category confidence logic:**

| Classification | High | Medium | Low |
|---------------|------|--------|-----|
| `chrom_assigned` | Gene proportion ‚â•20% AND GC deviation <2œÉ | Gene proportion 10-20% OR GC deviation 2-3œÉ | Gene proportion <10% OR GC deviation >3œÉ |
| `chrom_unassigned` | ‚Äî | GC deviation <2œÉ | GC deviation ‚â•2œÉ |
| `organelle_complete` | Coverage ‚â•90% | Coverage 80-90% | ‚Äî |
| `organelle_debris` | ‚Äî | Coverage ‚â•60% | Coverage <60% |
| `rDNA` | Coverage ‚â•80% AND identity ‚â•95% | Coverage 50-80% OR identity <95% | Coverage <60% |
| `contaminant` | Coverage ‚â•80% OR GC deviation >2œÉ | Coverage 50-80% | Coverage <50% |
| `chrom_debris` | Coverage ‚â•90%, identity ‚â•95%, GC deviation <3œÉ | Coverage 80-90% OR identity 90-95% OR GC deviation ‚â•3œÉ | ‚Äî |
| `debris` | Coverage ‚â•80% OR ‚â•5 protein hits | Coverage 50-80% OR 2-5 protein hits | GC deviation >3œÉ AND no synteny |
| `unclassified` | ‚Äî | ‚Äî | Always (no evidence) |

### Assignment Status Values

| Status | Description |
|--------|-------------|
| `OK` | Contig passed all assignment criteria |
| `NO_HITS` | No synteny evidence found |
| `AMBIG_LOW_FRAC` | Best reference coverage below `--assign-min-frac` threshold |
| `AMBIG_LOW_RATIO` | Score ratio below `--assign-min-ratio` threshold |

## contaminants.tsv

Detailed contaminant summary with full taxonomic lineage. Generated only when contaminants are detected. Used for phylogenetic breakdown visualizations (treemap and summary table).

| Column | Type | Description |
|--------|------|-------------|
| `contig` | string | Contig name |
| `length` | integer | Contig length in base pairs |
| `taxid` | integer | NCBI taxonomy ID of best centrifuger classification hit |
| `sci_name` | string | Scientific name of organism |
| `score` | integer | Centrifuger score (roughly proportional to matching sequence length; score ~1000 indicates ~1kb match with k=31) |
| `coverage` | float | Fraction of contig covered by taxonomic classification (0.0-1.0) |
| `kingdom` | string | Kingdom (Domain for Bacteria/Archaea/Viruses, or Cavalier-Smith kingdom for eukaryotes: Animalia, Plantae, Fungi, Chromista, Protozoa) |
| `phylum` | string | Phylum |
| `class` | string | Class |
| `order` | string | Order |
| `family` | string | Family |
| `genus` | string | Genus |
| `species` | string | Species |
| `depth_mean` | float | Mean read depth (if `--reads` provided) |
| `depth_median` | float | Median read depth (if `--reads` provided) |

**Notes:**

- Taxonomic lineage fields (kingdom through species) are populated via taxonkit if available. If taxonkit is not installed, genus and species may be parsed from scientific names, while higher-level taxonomy will be empty.
- The `kingdom` field uses a hybrid approach: Bacteria, Archaea, and Viruses use domain-level classification, while eukaryotes are mapped to Cavalier-Smith kingdoms (Animalia, Plantae, Fungi, Chromista, Protozoa) for more biologically meaningful grouping.
- Coverage filtering: Contaminants with coverage below `--contaminant-min-coverage` (default 0.50) are excluded from classification as contaminants but may still appear in this file if they pass the score threshold. Low-coverage hits may represent conserved genes rather than true contamination.

### Contaminant Table Visualization (HTML)

When contaminants are detected and `--plot` is enabled, final_finalizer generates an interactive HTML table (`*.contaminant_table.html`) summarizing the top contaminants ranked by abundance.

**File:** `*.contaminant_table.html`

**Display Format:**

The table shows the top 10 contaminants (configurable) with the following columns:

| Column | Description |
|--------|-------------|
| Rank | Numeric rank by abundance (depth √ó length if depth data available, otherwise total length) |
| Domain | Three-letter domain/kingdom abbreviation displayed as a colored badge (Bac=Bacteria, Arc=Archaea, Vir=Viruses, Ani=Animalia, Pla=Plantae, Fun=Fungi, Chr=Chromista, Pro=Protozoa) |
| Family | Taxonomic family name with colored left border for visual grouping |
| Species | Binomial species name (genus + species) in italics; shows "genus sp." if species unknown |
| n | Number of contigs assigned to this species |
| Total Mb | Total sequence length in megabases with inline gradient bar proportional to abundance. For multi-contig entries, shows aggregate total plus individual contig size range in parentheses (e.g., "5.23 Mb (2.10‚Äì3.13)") |
| Depth | Mean read depth with inline gradient bar (only if `--reads` provided). For multi-contig entries, shows mean depth plus min‚Äìmax range across individual contigs (e.g., "45√ó (30‚Äì60)") |
| Cov% | Mean coverage percentage across all contigs of this species |
| Circ | Circular topology indicator: ‚óè = all circularized, ‚óê = mixed, ‚óã = all linear |

**Ranking Logic:**

Contaminants are aggregated by binomial species name and ranked by abundance:
- **With read depth data** (`--reads` provided): Abundance = sum of (depth √ó length) across all contigs of each species. This prioritizes abundant, high-coverage contaminants that represent true contamination events.
- **Without read depth data**: Abundance = total length. Shows contaminants ordered by total sequence span.

**Aggregation and Spread Values:**

When multiple contigs map to the same species, the table aggregates data and shows spread values in parentheses:
- **Total Mb spread**: Highlights large individual contigs that may represent complete genomes
- **Depth spread**: Reveals depth variability across contigs, which can indicate:
  - Copy number variation (high depth = multi-copy plasmids or genomic integrations)
  - Assembly quality differences (depth variance suggests misassemblies or chimeras)
  - Mixed contamination sources (different strains/isolates)

**Visual Features:**

- **Inline gradient bars**: CSS gradient backgrounds provide quick visual comparison within each column (Total Mb and Depth)
- **Domain badges**: Colored labels allow rapid identification of major contamination sources (bacterial, fungal, viral, etc.)
- **Family color coding**: Consistent colored borders group related organisms across the table
- **Binomial italics**: Standard biological nomenclature formatting
- **Footer legend**: Shows all detected families with their color assignments

**Interpretation Guidelines:**

- **Large total Mb with high depth**: Likely represents substantial contamination from a co-purified organism or environmental source
- **High contig count with low individual Mb**: Suggests fragmented contamination (incomplete assembly or low-abundance source)
- **High depth variance**: May indicate mixed contamination sources, plasmids, or technical artifacts
- **Circular contigs**: Often represent complete organellar genomes (mitochondria, chloroplasts) or bacterial plasmids
- **Low coverage percentage**: May represent conserved genes rather than true contamination (these should be filtered by `--contaminant-min-coverage` during classification)

**Technical Notes:**

- Output is HTML-only (not PDF) because inline gradient bars require CSS rendering
- The HTML file can be opened in any web browser and printed to PDF if needed
- The table uses the R `gt` package for formatting
- Top N contaminants can be configured in the source code (default: 10)
- Table is skipped if no contaminants detected or if Rscript/required R packages are unavailable

## Metrics Calculation Details

This section explains how key metrics are computed from alignment data.

### Alignment Identity

The tool computes alignment identity as `matches / aln_len` from PAF alignment fields:
- **matches** (PAF column 10): Number of matching bases in the alignment
- **aln_len** (PAF column 11): Alignment block length (includes matches + mismatches + gaps)

This is an **alignment-weighted identity** that:
- Aggregates across all chains per (contig, reference) pair by summing matches and aln_len separately
- Differs from traditional BLAST-style percent identity because it's computed from minimap2/miniprot's internal alignment scoring
- Provides a robust metric for comparing synteny strength across chromosome sets

**Mode differences:**
- **Nucleotide mode**: Represents true sequence-level identity from whole-genome alignment
- **Protein mode**: Reflects protein alignment identity mapped to query genomic coordinates (miniprot aligns proteins to genome, so this is not protein-to-protein identity)

The `best_identity` column in `contig_summary.tsv` is the identity to the assigned reference chromosome, computed as `best_matches / best_aln_len`.

### Union Base Pairs (union_bp)

The `union_bp` metric represents non-overlapping base pairs on the query contig covered by alignments to a reference chromosome. It is computed by:

1. Collecting all alignment intervals `[start, end)` for a (contig, reference) pair
2. Merging overlapping intervals into non-overlapping regions
3. Summing the lengths of merged intervals

This measures the **breadth of alignment coverage** rather than raw alignment length, preventing double-counting of overlapping alignments.

### Chain Scoring (score_topk vs score_all)

Two scoring aggregations are provided for each (contig, reference) pair:

| Metric | Description | Use Case |
|--------|-------------|----------|
| `score_topk` | Sum of top-K chain scores (K = `--assign-chain-topk`, default 3) | Focuses on strongest evidence; reduces impact of noisy secondary chains |
| `score_all` | Sum of ALL chain scores | Considers total cumulative evidence; better for fragmented assemblies |

The `--assign-ref-score` parameter controls which is used for chromosome assignment:
- `topk`: Use `score_topk` (focuses on best chains)
- `all` (default): Use `score_all` (considers all evidence)

Individual chain scores are computed using the formula specified by `--assign-chain-score`:
- `matches` (default): Total matching bases in chain
- `qbp_ident`: Query base pairs √ó identity (balances coverage and quality)
- `matches_ident`: Matching bases √ó identity (most stringent)

## Usage Tips

### Filtering for High-Confidence Assignments

```bash
# Get chromosome contigs with status OK
awk -F'\t' 'NR==1 || ($3=="chrom" && $10=="OK")' output.contig_summary.tsv

# Get contigs with high gene density (>50 genes/Mbp)
awk -F'\t' 'NR==1 || ($18 > 50)' output.contig_summary.tsv
```

### Identifying Problematic Contigs

```bash
# Find chimeric contigs
awk -F'\t' 'NR==1 || $21=="yes"' output.contig_summary.tsv

# Find ambiguous assignments
awk -F'\t' 'NR==1 || $10 ~ /AMBIG/' output.contig_summary.tsv
```

### Analyzing Synteny Evidence

```bash
# Get all evidence for a specific contig
awk -F'\t' '$1=="contig_name"' output.evidence_summary.tsv

# Find contigs with evidence to multiple subgenomes
awk -F'\t' 'NR>1 {print $1}' output.evidence_summary.tsv | sort | uniq -c | awk '$1>1'
```
